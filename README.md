## 🏗️ System Architecture

The pipeline decouples raw human queries into verifiable database transactions using a multi-layered agent design:

```
[User Question] ──> [FastAPI Routing] ──> [Llama 3.3 70B Decomposition]
                                                     │
[NL Summary] <── [Summary Engine] <── [SQL Execution Loop] <── [Security Validator]
                                            │ (If Driver Fails)
                                            └─> [3x Self-Healing Prompt] ──┘

```

* **Decomposition Engine:** Consumes raw text questions and utilizes `llama-3.3-70b-versatile` to dynamically map schemas, determine filtering scopes, and draft relational table joins.
* **Security Isolation Layer (`validator.py`):** Intercepts generated queries using regex firewalls to enforce a strict read-only state. Any non-`SELECT` statements (`DROP`, `DELETE`, `UPDATE`, etc.) trigger an immediate security abort.
* **Autonomous Error Healing (`executor.py`):** Captures native PostgreSQL engine compiler errors. If a query fails, the agent parses the driver's traceback error message along with the database diagnostic hints and auto-reissues a repaired syntax draft (up to 3 structural attempts).
* **Insight Synthesis Engine:** Converts raw tuple lists into conversational, single-sentence business analyst answers.

## ⚙️ Prerequisites & Environment Setup

### 1. Clone the Repository

```bash
git clone https://github.com/Sadip08/Text-to-SQL-Agentic-Pipeline.git
cd Text-to-SQL-Agentic-Pipeline

```

### 2. Configure the Python Environment

```bash
# Create a virtual environment
python -m venv venv

# Activate the virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install core dependencies
pip install fastapi uvicorn groq psycopg2-binary python-dotenv pydantic requests

```

### 3. Establish Local Environment Keys

Create a `.env` file in the root directory to map your local PostgreSQL database container and production Groq inference keys:

```env
GROQ_API_KEY=gsk_your_production_secure_api_key_here
DB_HOST=127.0.0.1
DB_NAME=your_database_name
DB_USER=your_postgres_user
DB_PASSWORD=your_secure_password
DB_PORT=5432

```

### 4. Run the Server Instance

```bash
python main.py

```

The documentation dashboard will initialize locally at `http://127.0.0.1:8000/docs`.

## 🚀 API Reference & Endpoint Contracts

### Task 4: Autonomous Agent Endpoint (`POST /agent/sql`)

Accepts a single raw string text block. All intent parsing, indexing, multi-step error recovery, and summarizations occur under the hood.

* **Request Body:**

```json
{
  "question": "How many shipped orders are from USA customers?"
}

```

* **Successful Response Payload (200 OK):**

```json
{
  "sql": "SELECT COUNT(*) FROM \"orders\" JOIN \"customers\" ON \"orders\".\"customerNumber\" = \"customers\".\"customerNumber\" WHERE \"orders\".\"status\" = 'Shipped' AND \"customers\".\"country\" = 'USA';",
  "result": [
    {
      "count": 42
    }
  ],
  "summary": "There are 42 shipped orders from customers in USA.",
  "status": "success"
}

```

---

### Task 3: Structured Pipeline Endpoint (`POST /api/v1/execute-text2sql`)

A backward-compatible transaction endpoint that accepts pre-decomposed query attributes from client forms.

* **Request Body:**

```json
{
  "question": "How many shipped orders are from USA customers?",
  "Intent": "Count shipped orders from USA",
  "Tables": ["orders", "customers"],
  "Columns": ["status", "country", "customerNumber"],
  "Filters": "orders.status = 'Shipped' AND customers.country = 'USA'",
  "Joins": ["orders.customerNumber = customers.customerNumber"]
}

```

* **Response Payload (200 OK):**

```json
{
  "question": "How many shipped orders are from USA customers?",
  "sql": "SELECT COUNT(*) FROM \"orders\" JOIN \"customers\" ON \"orders\".\"customerNumber\" = \"customers\".\"customerNumber\" WHERE \"orders\".\"status\" = 'Shipped' AND \"customers\".\"country\" = 'USA';",
  "result": [{"count": 42}],
  "summary": "There are 42 shipped orders from customers in USA.",
  "status": "success",
  "meta": {
    "retry_applied": false,
    "error_log": null
  }
}

```
